## Plan: Google Play Billing Prep Refactor

Refactor subskrypcji pod Google Play Billing na Androidzie z weryfikacja w Supabase Edge Functions; Supabase `users` zostaje zrodlem prawdy dla FeatureAccessService; inne platformy zostaja stub.

**Steps**
1. Zdefiniuj mapowanie `SubscriptionPlan` -> Google Play product IDs + package name; wybierz miejsce config (appsettings / Resources/Raw / constants) i waliduj na starcie. *depends on answers*
2. Dodaj zaleznosc billing + wrapper klienta wg Billing_Documentation.md: minimalny interfejs (init, query products, purchase, restore/query owned, acknowledge). *depends on 1*
3. Rozszerz `SubscriptionOperationEntry` o provider dane: provider name, productId, purchaseToken, purchaseTime, expiry, acknowledged, last provider status; dodaj migracje. *depends on 1*
4. Zbuduj Supabase Edge Functions z jasna logika body.
   - verify-play-subscription: input supabaseUserId, productId, purchaseToken, packageName. Logika: walidacja; `purchases.subscriptions.get`; weryfikacja purchaseState/paymentState/expiryTime; mapowanie productId -> plan; `purchases.subscriptions.acknowledge` gdy potrzeba; update users (IsPremium, PremiumFrom, PremiumTo, UserType); response isValid/plan/premiumFrom/premiumTo/userType/acknowledged/message.
   - sync-play-subscription: input supabaseUserId, packageName, purchases[] (productId, purchaseToken). Logika: petla po tokenach, `purchases.subscriptions.get`, wybor najlepszej aktywnej subskrypcji po expiry, acknowledge gdy potrzeba, update users, response per-item status + final plan.
   - Supabase setup: wlacz Android Publisher API, service account, sekrety (service account JSON, package name, opcjonalne productIds), service-role key do update users, spisz env vars. *depends on 1*
5. Zaimplementuj `PaymentProviderSubscriptionManagementService` dla Android:
   - init billing, query products,
   - po purchase -> verify-play-subscription -> update users -> operacja Completed,
   - przy bledach/network -> Pending + zapis tokenu,
   - ResumePendingOperationAsync ponownie wola verify-play-subscription,
   - CancelSubscriptionAsync przekierowuje do Google Play manage subscription + refresh dostepu. *depends on 2, 3, 4*
6. Zaktualizuj UI subskrypcji w ProfilePage.xaml i ProfilePage.xaml.cs:
   - pokaz status weryfikacji/pending z `SubscriptionOperationStatus`,
   - cancel button na Android: “Manage subscription in Google Play,”
   - opcjonalnie live price/period z Google Play product details. *depends on 5*
7. DI: Android -> `PaymentProviderSubscriptionManagementService`, reszta -> `MockSubscriptionManagementService`. *depends on 5*
8. Dopisz projektowy opis flow i Supabase setup do Billing_Documentation.md. *parallel with 6*

1. Android purchase monthly: users row update (IsPremium true, PremiumFrom/To), UI -> Premium.
2. Network fail: operacja Pending, Resume dziala po powrocie netu.
3. Restore: query owned, sync-play-subscription, entitlements update.
4. Cancel: manage subscription link -> po anulowaniu backend czysci Premium, UI -> Free.
